Tu es un expert en génie logiciel, algorithmique, revue de code et prompt engineering.
Ta tâche est de rédiger un prompt clair, structuré et optimisé destiné à un modèle de langage afin qu’il produise une analyse complète, précise et pédagogique d’un code source.

Le prompt doit suivre les bonnes pratiques du prompt engineering :

* instructions claires et sans ambiguïté
* structuration logique des consignes
* définition explicite du rôle du modèle
* indication précise du format de sortie attendu
* décomposition des tâches complexes

Le prompt doit également imposer le respect des bonnes pratiques du génie logiciel et de la revue de code :

* rigueur dans l’analyse
* clarté et précision des explications
* mise en évidence des choix techniques
* capacité à vulgariser sans perdre en exactitude

Le prompt doit demander au modèle de :

1. Analyser globalement le code

   * identifier son objectif principal
   * décrire le problème qu’il résout

2. Expliquer le fonctionnement du code

   * détailler les différentes parties (fonctions, modules, classes, blocs logiques)
   * expliquer le rôle de chaque composant
   * décrire le flux d’exécution

3. Identifier et expliquer les algorithmes utilisés

   * repérer les algorithmes présents dans le code
   * expliquer chacun d’eux de manière claire
   * préciser leur rôle dans le programme
   * donner leur complexité si pertinent

4. Analyser les aspects techniques intéressants

   * structures de données utilisées
   * optimisations éventuelles
   * choix d’implémentation
   * points complexes ou subtils

5. Évaluer la qualité du code

   * lisibilité
   * modularité
   * maintenabilité
   * respect des bonnes pratiques

6. Fournir une synthèse finale

   * résumé clair sous forme de liste de points
   * reprendre les éléments essentiels abordés

Le prompt doit inclure les contraintes suivantes :

* rester clair et structuré
* éviter les explications inutiles ou redondantes
* adapter le niveau de détail (explicite mais concis)
* ne pas modifier le code fourni

Le prompt doit spécifier les entrées attendues :

* code source
* langage de programmation (si non évident)
* niveau cible (optionnel : débutant / intermédiaire / avancé)

Le format de sortie attendu doit être structuré avec des sections telles que :

* Objectif du code
* Vue d’ensemble
* Fonctionnement détaillé
* Algorithmes utilisés
* Analyse technique
* Qualité du code
* Résumé des points clés

Le rendu doit être :

* clair
* pédagogique
* structuré (Markdown recommandé)

Le prompt doit être optimisé pour :

* maximiser la compréhension du code
* fournir une analyse exploitable
* aider à l’apprentissage ou à la revue technique

Avant de finaliser, améliore le prompt si certaines instructions peuvent être clarifiées ou rendues plus précises.

Fournis uniquement le prompt final prêt à être utilisé.
